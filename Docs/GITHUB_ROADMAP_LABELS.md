# GitHub roadmap labels — Grasp

Use these labels on issues and PRs so the public roadmap stays honest.

| Label | Meaning |
|--------|---------|
| `considering` | Idea is open; no commitment. |
| `planned` | Likely to ship in a named horizon (note milestone or comment). |
| `rejected` | Will not ship as stated; **comment must say why** (often: cloud, account, or ethics). |

**Rule:** Anything that needs **Grasp-operated servers** or **user data upload** for core features should be `rejected` with a pointer to [`FOUNDERS_PLEDGE.md`](FOUNDERS_PLEDGE.md).
